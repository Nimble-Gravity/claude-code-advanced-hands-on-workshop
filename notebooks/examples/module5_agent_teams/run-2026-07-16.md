## Notes / anomalies

1. **Agent Teams did not activate under headless `claude -p`.** No
   `~/.claude/teams/` directory was ever created during the 494 s run; the
   lead's own summary states the teammates "had no messaging tool" and that
   it relayed every debate turn between them itself. Cell 19's sanity note
   (fewer team tokens usually means "the flag was off and Claude quietly used
   subagents") describes exactly this failure shape. The interactive-terminal
   instruction in Part A is load-bearing; the notebook could warn that `-p`
   silently degrades.
2. **Decisions did not tie.** Part D asserts "Both runs should have reached
   the same ESCALATE-shaped answer." Headless Part A decided REVISE and then
   implemented a transactional outbox, editing 4 files in `src/contoso/`
   (reverted after this test). The subagent control decided ESCALATE and
   edited nothing. The Part A prompt permits shipit to edit after consensus,
   so a REVISE-then-implement outcome is prompt-conformant.
3. **`reference/m5/sample-run/` is empty** except for a facilitator README
   that says to record it before the workshop. Attendees whose preflight
   fails currently have no fallback material.
4. **Part B's restore step depends on git state.** `git checkout --
   .claude/agents/skeptic.md` fails while the agent files are untracked (they
   are uncommitted as of this run); it only works once the module is
   committed.
5. Environment nit: macOS ships no `timeout(1)`; the 10-minute timeboxes used
   a shell watchdog instead. Not a notebook issue.
6. Cleanup performed: executed notebook copy deleted; source edits from the
   headless Part A run reverted via `git checkout -- src/contoso/`;
   `artifacts/m5/` left as the runs produced it (gitignored).

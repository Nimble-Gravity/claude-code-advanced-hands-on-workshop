## Key Learnings Demonstrated

1. **Criteria-first testing**: Tests derive from acceptance criteria, not from code reading. This makes tests able to actually fail.
2. **Vacuous-test trap**: A test touching `created_at` (never read) would still pass vacuously. Our tests touch `last_used_at` (the operative column).
3. **Coverage matrix**: Maps criterion → test → verdict (verified/vacuous/untested), explicitly naming residual risk instead of ending with "all green."
4. **Boundary cases**: The `<=` operator in TTL logic is explicitly tested with both sides (30 days valid, 30d+1s invalid).
